# Callouts v1.3.0

## VFX capture to file

A new opt-in tool for offline analysis: append every observed VFX spawn to a file,
with **no size or line limit**. Useful for finding the exact VFX paths that tell
"real" vs "fake" mechanics apart (or any path-level investigation).

- **Settings → VFX CAPTURE → "Capture VFX to file"** toggle (persisted).
- Writes `vfx-capture.log` in the plugin config folder, tab-separated:
  `time · territoryId · zone · target(self/party/other) · combat · path`.
  A session banner delimits each capture run.
- Append-only: the file grows until you turn capture off or use **Clear capture file**.
- Auto-flushed so nothing is lost if the game crashes mid-fight; any IO error
  disables capture rather than disturbing the event pipeline.

### Requirement
VFX events only flow while **advanced sources are enabled** and the **VFX hook
signature is configured** for your client. In a build where the signature is not
set (the default), advanced sources report FAILED and the capture file will only
contain session banners — no rows. The Settings panel states this inline.
