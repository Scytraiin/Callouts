# Callouts v1.4.0

## Timeline (Cactbot-style)

A live, on-screen timeline of upcoming mechanics with countdowns — plus a way to build one
from a fight you just did.

- **Live view** (⏱ Timeline button, `/callouts timeline`): the active timeline's elapsed
  clock and upcoming entries as shrinking countdown bars.
- **Sync / drift correction**: entries can carry a sync pattern (name-contains or regex);
  when that mechanic actually casts, the clock re-anchors to keep the countdowns honest.
- **Propose from last fight**: after a pull, generate a draft timeline from what was
  observed (enemy casts + debuffs on you/party), with sync patterns and times filled in —
  then edit it.
- **Editor**: per-entry time / label / sync / pre-alert lead / alert text / sound;
  enable per zone; auto-pick by zone or choose an active timeline manually.
- **Pre-alerts** fire N seconds early through the normal Echo / Toast / Sound sinks.
- **Import / export** a timeline as a `TL1|` clipboard code (gzip-bomb guarded).

## Debuff timer matching

Status rules can now match on the **applied timer**, so the SHORT and LONG version of the
same debuff can do different things (the classic short/long personal check).

- New **Min / Max timer sec** fields on status rules (0 = unbounded). E.g. 14–16 catches
  only the ~15 s version.
- New `{duration}` placeholder for Echo/Toast text.
- Only applies to *gained* statuses (a removal reports no timer).

## Not included (by design)
Auto-executing in-game macros / commands / actions (e.g. auto-marking) is intentionally
absent: it would require sending input to the game, which violates the game's ToS and
Dalamud's plugin rules. Callouts stays informational — it tells you; it never acts for you.
