# Callouts v1.0.1

Maintenance release: correctness and UX fixes from a code-review pass. No config
migration required from v1.0.0.

## Fixes
- **VFX / head-marker "party" scope** now works — party membership of the marked
  actor is actually resolved (was effectively never matching).
- **VFX / head-marker "target" scope** now requires the marked actor to be your
  target (previously it matched every actor).
- Rules with **no output enabled** no longer consume a cooldown or a rate-limit
  token (relevant to imported rule packs).
- **Cast / status** detection maps are cleared on zone change.
- **Status feedback** (export/import/delete/undo) is always visible, not only while
  the rule editor is open.
- The editor **toast "Test"** is now `[test]`-prefixed, like the row test-fire.
- The **rate-limit setting** applies immediately (no reload needed).
- The **advanced-source failure notice** shows once per session instead of on every
  toggle.
- **Cooldown** is validated to 0–600 s; **imported rules** are validated (invalid
  ones are skipped with a count); **test-fire** now fills `$1`..`$3`.
- The editor no longer steals keyboard focus for non-chat rules.
